### 分支操作

```shell
# 创建分支
git branch <分支名>
#切换废止
git checkout <分支名>
git switch <分支名>

#二合一
git checkout -b <name>
```

## 合并

```sh
git merge <目标>
```

### rebase

Rebase 实际上就是取出一系列的提交记录，“复制”它们，然后在另外一个地方逐个的放下去。

Rebase 的优势就是可以创造更线性的提交历史，这听上去有些难以理解。如果只允许使用 Rebase 的话，代码库的提交历史将会变得异常清晰。

**将当前分支复制一份加到目标上**

```sh
git rebase <目标>
```
